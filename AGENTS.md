# AGENTS.md

## Git commits
- keep commit messages brief
- use lower-case only
- avoid prefixes like `feat:` or `fix:`
- target length: 3-8 words

## Editing Constraints
- Make the minimal set of changes required to solve the problem.
- Do not modify unrelated code or refactor existing logic unless required for correctness, clarity, or safety.
- Prefer localized changes; avoid touching multiple files or functions unless necessary.
- If a change expands beyond the immediate scope, pause and explain why before proceeding.

## Tests and examples
- use generic placeholder names in tests, fixtures, factories, seeds, and examples
- prefer names like `Test User` instead of real personal names
